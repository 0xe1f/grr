    grr >:(

    grr is a lightweight Google Reader replacement, supporting many primary features of Reader with support for multiple users.

    grr consists of a web server application, which interacts with the user, and a shell application, which is routinely executed to update feeds in the background.

    grr is currently in a late alpha/early beta stage.

    Features

    • Feed hierarchy (nesting feeds inside folders)
    • Starring
    • Marking as read/unread (single articles as well as entire views)
    • Filtering by status (All/Unread/Starred)
    • Paging
    • Support for multiple users

    Requirements

    On the server side, grr requires:

    • Web server with PHP support
    • MySQL server
    • cron, or any other scheduling service to routinely update feed information

    On the client side, grr relies heavily on JavaScript and probably a decent modern browser. Authentication requires an OpenID account (any OpenID account).
